# 入网套餐后台管理系统 PRD (草案)

## 1. 目标
- 做一个“后台记账系统”性质的管理后台，支持管理员和代理两类角色。
- 代理支持分级与上下级关系（至少支持查看一级/二级下级）。
- 系统能够按固定周期自动计算佣金，并输出可审计、可复算的结算明细。

## 2. 角色与权限

### 2.1 管理员 (Admin)
拥有全量管理权限，主要负责“配置 + 数据录入/变更 + 结算”。
- 组织与人员
  - 团队管理：新增/编辑团队标签；管理团队成员（必须能带上团队成员名字）
  - 员工/代理管理：新增/编辑基本信息（姓名、手机号、工号、签约星级、所属团队、所属渠道、所属省份等）
  - 代理上下级关系维护（避免环/重复）
- 业务配置
  - 星级等级管理：新增/编辑等级、等级说明、扶持期/稳定期、佣金比例、结算周期等
  - 套餐管理：套餐名称、月租（佣金计算基数）、成本/公司价等
  - 政策管理：政策版本化（生效时间、适用范围、关联等级规则等）
  - 渠道管理：渠道公司/名称，渠道下代理归属
- 运营数据
  - 网卡管理：录入网卡（卡号、入网/激活日期、套餐、政策、归属代理、状态）
  - 状态管理：在网/停机/离网/异常等，状态变更需记录原因与时间
- 结算
  - 结算跑批：每月 5 号自动结算上月佣金；支持按代理筛选并重新计算预览
  - 审核与入账：结算结果审批后入账（生成记账分录）
  - 调整：发现错误用“调整单”修正，禁止直接改历史结算明细

### 2.2 代理 (Agent)
只能访问自身与下级范围内的数据。
- 我的同事
  - 查看一级/二级下级成员
  - 显示下级等级、下级网卡数量（尤其在网 5G 卡）
  - 显示等级差价/团队业绩（以结算明细为准）

团队展示（Agent 端）：
- 只显示：团队成员姓名、在网号卡数量、星级
- 我的 5G 网卡
  - 列表：在网卡号(按隐私规则脱敏)、激活日期、号卡对应政策
  - 我的星级：当前星级、到期时间/维持任务进度（如有）
- 公告
  - 查看后台公告

## 2.3 隐私与展示规则 (Agent 端)
- 号码显示：
  - 代理查看“自己开的卡”：显示完整卡号
  - 代理查看“团队/下级的卡”：卡号自动隐藏，示例：`192******16`
- 团队标识展示：
  - 文案固定为“团队”两个字，不可改
  - “团队：{姓名}”中的姓名自动取该卡实际归属/开卡成员姓名

## 3. 核心业务对象
- 团队 (Team)：标签/名称、成员列表；团队变更必须可追溯
- 代理 (Agent)：基本信息、所属团队/渠道/省份、星级、上下级关系
- 星级等级 (AgentLevel)：扶持期/稳定期规则、佣金比例（按期别）、结算周期
- 套餐 (Plan)：月租（佣金基数），其他价格字段可选
- 政策 (Policy)：佣金规则的“版本化载体”，用于结算可复算
- 网卡 (Card)：卡号、激活/入网日期、套餐、政策、归属代理、状态
- 结算单 (SettlementRun)：结算月、使用的政策版本、状态（草稿/已审核/已入账）
- 结算明细 (SettlementItem)：按“卡 x 收益人(代理) x 月”生成行项目
- 账本/分录 (LedgerEntry)：结算入账后的会计式流水（便于对账）

## 4. 关键流程

### 4.1 配置流程
1. 管理员创建/维护星级等级与佣金比例（必须版本化或至少记录生效时间）
2. 管理员维护套餐与政策（政策关联等级规则，避免散落配置）
3. 管理员维护团队与成员（支持带上成员名字；否则无法新增或管理团队）

团队补充口径：
- 代理只能归属到一个团队（同一时间只能有一个有效团队归属）
- 团队标签/名称允许自定义（创建者或团队队长可编辑）

### 4.2 录入流程
1. 管理员录入网卡：选择政策、归属代理、套餐、激活日期
2. 后续卡状态变化（停机/离网/异常）按事件记录

### 4.3 结算流程（按月）
1. 结算跑批生成草稿（可只选某个代理预览）
2. 生成“按卡明细”与“按代理汇总”
3. 管理员审核后入账（锁定结算单）
4. 如需修正：创建调整单（反冲+补记），保留审计链路

结算口径（已确认）：
- 扶持期固定 11 个月（所有星级一致）
- 激活月不计佣；从激活次月开始计佣
- 扶持期内可结算月份为激活后第 2..11 月；之后进入稳定期
- 稳定期有效月份按星级配置；到期后该卡不再结算佣金
- 当月出现任何异常状态，则当月完全不结算佣金
- 金额保留 2 位小数，2 位后的直接去掉（截断）

## 5. 非功能性要求
- 可复算：任何历史月份结算，必须能在不依赖“当前配置”的情况下解释结果
- 幂等：同一结算月重复执行不应重复入账
- 审计：关键表（团队/等级/政策/网卡状态/结算）都有操作人、时间、变更前后
- 性能：支持按代理查看全量卡与下级数据（至少支持分页+索引优化）

## 6. MVP 建议
第一版先把“能结算、能查明细、能审计”跑通：
- Admin：团队管理、代理管理、等级/政策/套餐、网卡录入与状态、按月结算预览+入账
- Agent：我的同事、我的 5G 网卡、公告
