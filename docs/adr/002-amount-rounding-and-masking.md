# ADR-002: 金额取整与号码脱敏规则

## 状态
- Accepted (已确认)

## 1. 金额取整
### 决策
- 所有结算行项目金额保留小数点后 2 位，2 位后的直接去掉（截断，不四舍五入）。

### 原因
- 口径明确要求“2 位后的直接去掉”，且示例场景以月租 * 比例计算，截断规则容易实现且可测试。

### 影响
- 金额计算统一使用 `trunc2()`，避免不同语言/ORM 的浮点误差导致对账不一致。
- 强制使用 decimal/num 类型（或整数分单位计算）并写入黄金测试用例。

## 2. 号码脱敏 (Agent 端)
### 决策
- 数据库存完整卡号。
- 代理端只允许“自己开的卡”展示完整号，其他卡号统一脱敏显示。
- 脱敏格式示例：`192******16`（保留前 3 位 + 6 个星号 + 后 2 位；不足位数按统一规则降级处理）。

### 原因
- 团队/下级卡号需要隐私隔离，避免横向泄露。

### 影响
- API 层需要根据 viewer_id 与 card_owner_id 决定返回 `card_no_full` 或 `card_no_masked`。
- 前端不得自行拼接完整号；完整号只应由后端返回给有权限的人。
