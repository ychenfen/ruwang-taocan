# 自动化运行报告（2026-03-01）

## 本次目标
- 处理高优先级佣金一致性缺陷：
  - 状态事件“发生日期”与实际结算口径不一致（时区/日期偏移）
  - 网卡转移归属缺少“生效日期”导致回算口径不清晰
  - 结算重算中关系回退到当前关系导致历史月不稳定

## 代码变更
- 后端 `backend/src/routes/admin/cards.ts`
  - `POST /admin/cards/:id/assign` 新增 `effectiveAt (YYYY-MM-DD)` 入参。
  - 转移时旧 assignment `end_at` 与新 assignment `start_at` 均按 `effectiveAt`（默认当前时刻）写入。
  - `POST /admin/cards/:id/status-events` 的 `happenedAt` 支持 `YYYY-MM-DD`，统一规范化为 `YYYY-MM-DDT00:00:00+08:00`。
- 结算计算 `backend/src/settlement/compute.ts`
  - 月内状态窗口改为固定 `+08:00` 月起止，避免服务器本地时区差异导致边界偏移。
  - 上下级关系口径改为严格按结算月月末快照，不再回退到当前关系，确保历史可复算。
- 管理端 `admin-web/app/(app)/cards/[cardId]/page.tsx`
  - 转移归属新增“生效日期”输入，并提交 `effectiveAt`。
  - 日期展示改为本地年月日拼接，避免 `toISOString().slice(0,10)` 造成前一天显示。
- 管理端 `admin-web/app/(app)/settlements/[runId]/page.tsx`
  - 元信息新增“结算月份”，并将原“运行月份”标注为“运行月份(创建月)”。
- 测试更新
  - `backend/src/routes/admin/cards.test.ts`：覆盖 `effectiveAt` 与 `YYYY-MM-DD` 状态事件规范化。
  - `backend/src/routes/admin/settlements.test.ts`：更新为历史快照口径断言，并验证无历史关系时不回退当前关系。
  - `backend/src/routes/admin/reports.test.ts`：按历史快照口径更新断言。

## 影响说明
- 修复后，“状态发生日期=3/1”将稳定按 `+08:00` 口径入库，不再因 UTC 显示偏移误判到前一天。
- 历史月重算不会再被“当前上下级关系”污染，结算可复算性增强。
- 转移归属可指定生效日，减少“从哪月开始按新归属算”的歧义。
