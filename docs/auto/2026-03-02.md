### 用户反馈
- 佣金口径异常：`C <- B <- A` 场景下，`B` 与 `C` 同/近级无差价可接受，但 `A` 应有二级差价；同时“当月异常应不结算”需严格生效。
- 网卡详情“转移归属代理”报错：`POST /admin/cards/:id/assign` 返回 500。

### 数据核查（线上 API 实测）
- 目标卡：`8f7c6d3d-cf14-4fc0-91a1-f0fd4a307aca`
- 状态事件：
  - `PAUSED @ 2026-03-01T16:00:00.000Z`
  - `NORMAL @ 2026-03-01T16:00:00.000Z`
  - `NORMAL @ 2026-03-02T16:00:00.000Z`
- 结算探针：
  - 2026-03 重算：该卡命中行 `0`（当月异常拦截生效）
  - 2026-02 重算：该卡仅 `SELF`（收益人 C），无 `UPLINE_DIFF_2`
- 探针清理：
  - 已删除本次排查生成的 3 个草稿结算单（`ff5bb0b5-*`、`97e7192a-*`、`63676ea5-*`），避免污染线上数据。
- 关键结论：
  - 佣金引擎“异常月归零”逻辑生效；
  - 2026-02 无 A 差价的直接原因是该月快照未命中可分润上级链路（历史月按月末快照，不按当前实时关系回填）。

### 500 根因（已定位）
- backend 日志明确报错：

2. 代理上下级接口增强（`backend/src/routes/admin/agents.ts`）
   - `PUT /admin/agents/:id/upline` 新增 `effectiveAt(YYYY-MM-DD)`。

     - 开始前重写（rewrite before start）
